Changelog

What we've shipped.

Every release is documented here. Latest on top. We aim for one focused release every 1–2 weeks.

v1.0 Apr 26 · 2026
Launch

Initial SaaS launch.

PrivateForge v1.0 — the first public release. The full self-hosted agent platform with multi-tenant orgs, BYOK provider routing, billing, and the browser + sandbox runtimes.

  • Multi-tenant orgs. Real org/team/workspace primitives. Roles: owner, admin, member, viewer. Magic-link sign-in.
  • BYOK provider routing. Per-org keys for Anthropic, OpenAI, and Google — encrypted with AWS KMS envelope encryption. Calls go direct to the provider.
  • Billing & subscriptions. Stripe-backed billing, prorated upgrades, 7-day trial on every plan, per-user seat add-ons.
  • Browser runtime. Stagehand + Browserbase wired in. Agent can navigate, fill forms, and produce screenshot artifacts.
  • Code sandbox. E2B integration for isolated cloud micro-VMs — for risky tool calls and untrusted scripts.
  • Project memory. Per-project semantic memory. The agent remembers conventions and preferences across sessions.
  • History timeline. Searchable record of every action, plan, and tool call. Replayable runs.
  • Cost dashboard. Per-user, per-project, per-provider call accounting.
v0.9 Apr 12 · 2026
Beta

Closed beta — last polish before GA.

Two weeks of dogfooding across our 14 design-partner teams. Bug fixes, performance work, and the final pricing model.

  • Hardened the KMS decryption path; secrets now decrypt only at agent runtime, never on our servers.
  • Browser agent now resumes mid-step on flaky network — fixed long-tail failures on slow staging environments.
  • Memory writes batched — 4× fewer disk syncs during long agent runs.
  • Pricing finalized: $20 single, $50 team (5 users), $10/extra user.
v0.5 Mar 15 · 2026
Security

Private alpha.

First version handed to design partners. The architecture solidified here — agent runtime as a binary, control plane as a thin SaaS, BYOK with KMS envelope encryption.

  • Initial multi-provider routing (Anthropic, OpenAI).
  • Filesystem and shell tools.
  • Basic project memory.
