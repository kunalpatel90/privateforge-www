PrivateForge — Changelog Today v1.0.2

Every release.
Every meaningful change.

Versioned. Dated. Honest. We do not ship dark patterns and we do not ship them quietly.

  1. v1.0.2 Apr 28, 2026 Patch
    • Local app health probe now detects Claude Code v2.1.x credentials in macOS Keychain. Earlier versions only checked ~/.claude/.credentials.json, which the new Claude CLI no longer writes.
    • Login watcher polls the Keychain alongside the legacy file path so OAuth completions are detected on both old and new CLI versions.
    • Email displayed in Providers UI now sourced from ~/.claude.json when the credential lives in Keychain.
  2. v1.0.1 Apr 21, 2026 Patch
    • Stability fixes for long-running scheduled tasks on macOS Sonoma.
    • Fixed an issue where the audit log could miss entries written during system sleep.
    • Smoothed the cold-start latency of the Build station from ~1.4s to ~0.6s.
  3. v1.0.0 Apr 14, 2026 First public release
    • Six stations on the floor: Research, Build, Automate, Connect, Memory, Privacy.
    • Cloud providers: Anthropic Claude, OpenAI ChatGPT, Google Gemini. Bring your own keys.
    • Local model support via llama.cpp / MLX / Ollama. Any GGUF.
    • Ten connectors at launch: Gmail, Outlook, Zoho Mail, Google Calendar, Drive, GitHub, Notion, Linear, Slack, Stripe.
    • Durable, searchable, editable memory. Stored locally.
    • Append-only, replayable, exportable audit log.
    • End-to-end encrypted optional cloud sync (off by default).
  4. v0.9 Mar 12, 2026 Public beta
    • Added the Automate station with cron-style scheduling in plain English.
    • Added durable memory: facts about you and your projects, searchable and editable.
    • Audit log replay: rewind any past run step by step, with the exact tool inputs and outputs.
    • Connector permission scoping reduced to the narrowest set that works per-feature.
  5. v0.5 Jan 21, 2026 Closed alpha
    • Research and Build stations only. ~80 invited users.
    • Single cloud provider (Claude). Local models via Ollama.
    • Establishing the bench metaphor and the workshop UX. Naming was difficult.
