Step 1

Install

Download the installer for your platform. Run it. PrivateForge installs as a normal desktop app — no admin gymnastics, no daemons running in the background unless you want them.

  • macOS: Download the .pkg, double-click, follow the prompts. Requires macOS 13+.
  • Windows: Download the .msi, run the installer. Requires Windows 11.
  • Linux: A .deb and .AppImage are in beta — see changelog.

First launch takes about 8 seconds while the local agent runtime warms up. After that, cold start is under a second.

Step 2

Sign in to a provider

Open Settings → Providers. You can use any combination of cloud and local models. For your first run, sign in to whichever cloud provider you already pay for.

  • Anthropic Claude: Click Sign in. The app drives the Claude CLI through OAuth — your token is stored in the OS keychain.
  • OpenAI ChatGPT: Same flow, via the Codex CLI.
  • Google Gemini: Same flow, via the Gemini CLI.
  • Local models: Point at a running Ollama / llama.cpp / MLX endpoint. Bring your own GGUF.
Step 3

Run your first task

Open the workstation, type what you want, hit Enter. Treat it like a coworker who reads carefully and writes things down.

task  research the EV charging market and write a 1-pager
route claude-sonnet-4 · keys · local
tool  web.search  → 12 sources
tool  web.fetch   → bloomberg.com/ev-q1
write ~/forge/ev-1pager.md (1,842 words)
ok    done in 16.2s

Every step is visible. Every source is linked. Every file lands on your disk.

Research station

For pulling primary sources and synthesizing them. Configure default models in Settings → Stations → Research. Citation style, browser backend, and source whitelist are all editable.

Build station

For producing real files: PDFs, decks, sheets, web apps, scripts. The Build station writes to a project folder you choose. It will not write outside that folder unless you grant it permission for the run.

Automate station

Schedule recurring runs in plain English. "Every Friday at 4pm, summarize this week's gmail into 5 bullets and email me the result." The schedule lives locally; the run executes on your machine when it fires.

Connectors

Each connector authorizes via OAuth against your account, stores its token in the OS keychain, and can be revoked one-click from Settings → Connectors. Available: Gmail, Outlook, Zoho Mail, Google Calendar, Drive, GitHub, Notion, Linear, Slack, Stripe, Shopify, Zoho Books, plus any custom MCP server.

Memory

Durable, searchable, editable facts about you and your projects. Stored in a local SQLite file. Open Memory to read, edit, or delete entries. Export as JSON anytime.

Audit log

Every tool call, file write, and URL visit is recorded. Open History → Audit to replay any run step-by-step. Use --export-audit on the CLI to dump a JSON for compliance review.

CLI

Everything the desktop app does, the CLI can do too. Useful for cron, CI, and headless servers.

$ forge run "summarize ~/inbox/q3-research/*.pdf"
$ forge schedule --every "fri 16:00" --task "weekly review"
$ forge memory list --project acme
$ forge audit --since "yesterday" --json
