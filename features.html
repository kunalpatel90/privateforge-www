Features

An agentic workshop, behind your firewall.

Everything you'd ask of Cursor, Replit Agent, or v0 — and the parts they don't ship: per-org keys, role-based access, encryption with your KMS, full audit history.

Coding agent

A real coding workstation — running on your hardware.

Filesystem, shell, terminal, plan→execute loops, multi-file edits. Project memory that learns your conventions over time. Same loop as Claude Code, with one critical difference: every byte stays inside the box you booted it on.

  • Multi-file edits with diff preview
  • Persistent project memory and conventions
  • Long-horizon task plans you can pause and resume
  • Dev server tooling — npm, pip, cargo, go
Browser agent

A real browser. Pointed at the real web.

Stagehand and Browserbase under the hood. The agent navigates docs, fills forms, runs end-to-end tests against your staging environment, and screenshots failures back to you — without making you stand up Playwright by hand.

  • Headless Chromium with full DOM access
  • Auth-aware — log in to your tools, the agent picks up the session
  • Screenshot artifacts attached to every step
  • Recordable runs you can hand off as repro steps
BYOK · multi-provider

Your keys. Your providers. No proxy in the middle.

Anthropic, OpenAI, Google — bring any combination of API keys. The agent picks the right model for the task, falls back across providers, and tracks every dollar by user, project, and call.

  • Per-org keys, encrypted at rest with your KMS
  • Mix models inside one task (Claude for planning, Gemini Flash for tool calls)
  • Per-call cost tracking and provider-level rate limiting
  • No PrivateForge proxy — calls go direct to provider
# .privateforge/providers.toml
[anthropic]
api_key = "sk-ant-***"      # encrypted (KMS)
default_model = "claude-sonnet-4-5"

[openai]
api_key = "sk-***"            # encrypted (KMS)
default_model = "gpt-5"

[google]
api_key = "AIza***"           # encrypted (KMS)
default_model = "gemini-2.5-pro"

# LLM calls go DIRECT — never proxied through us.
Team & org model

An org, with structure. Not a shared login.

PrivateForge ships with a real organization model from day one: orgs, teams, workspaces, role-based access, invite flows, audit log. Stop pretending five people sharing a Cursor seat is a strategy.

  • Roles: owner, admin, member, viewer
  • Workspace-scoped projects, secrets, and history
  • Invite-by-email, magic-link sign-in, or SSO (coming soon)
  • Per-user usage and cost reports
Memory & history

The agent remembers. So can you.

Every conversation, plan, and tool call is recorded in a searchable history. Memory persists across sessions — the agent learns your repo's conventions and your team's preferences over time.

  • Per-project semantic memory
  • Searchable timeline of every action the agent took
  • Replayable runs — useful for evals and debugging
  • Export-friendly — your data is your data

Encryption & audit

KMS envelope encryption — with your CMK.

Every API key, OAuth token, and secret is encrypted at rest with AWS KMS envelope encryption. You bring your own customer master key. Decryption events are logged to your CloudTrail.

  • Envelope encryption with per-secret data keys
  • BYO CMK — keys never leave your AWS account
  • Decryption events visible in CloudTrail
  • Full audit log of every secret read
