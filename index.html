The problem

Every AI tool you use is a one-way mirror.

Cursor, Replit, v0, ChatGPT — they all want your code uploaded, your prompts logged, and your context routed through their servers. For a side project that's fine. For your actual work, it's a privacy and compliance nightmare. And for a regulated codebase, it's a non-starter.

Your code goes up.

Every file you open in a SaaS agent gets shipped to someone else's GPUs. Even with “zero retention” promises, your secrets cross the network.

Their key, their rules.

Most agents proxy LLM calls through their own infrastructure. They set the model, the rate limits, and — quietly — the training policy.

No team boundaries.

"Add three teammates" on most agentic tools means giving everyone the same anonymous bag of context. No roles. No audit. No org structure.

Compliance limbo.

Your security team asks "where does the data go?" You can't answer. Procurement freezes. The deal slips. You build with ChatGPT after hours.

The PrivateForge approach

Pull the AI to your data — not your data to the AI.

The agent runs as a binary you install. Your filesystem, your network, your KMS. Provider keys are encrypted at rest with your own AWS KMS — and never decrypted server-side.

How it works

From `git clone` to shipping in 4 minutes.

No infra to set up. No models to download. Install, sign in, paste an API key — and your private agent is ready.

What's in the workshop

A full agentic stack — without the SaaS dependency.

Everything Cursor, Replit, and v0 ship. Plus the parts they leave out: org structure, encryption-at-rest with your keys, and full observability.

Coding workstation

File system access, shell, terminal, project memory, plan→execute loops, multi-file edits. Same agent shape as Claude Code, with the data path inverted.

Browser automation

Stagehand + Browserbase wired in. The agent reads docs, files tickets, and tests your app in a real Chromium — without you wiring Playwright by hand.

Code sandboxes (E2B)

Spin up isolated cloud micro-VMs for risky operations — package installs, schema migrations, untrusted scripts — without polluting your machine.

BYOK across providers

Per-org keys for Anthropic, OpenAI, and Google. Mix models in a single project. Costs are tracked per call, per user, per provider.

Org · team · workspace

Real role-based access — owner, admin, member, viewer. Shared workspaces for the team. Private playgrounds for each user. No more shared logins.

KMS-encrypted secrets

AWS KMS envelope encryption for every API key, token, and secret. Your KMS, your CMK, your audit log.
