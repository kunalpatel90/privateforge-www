PrivateForge — Security posture Today v1.0

Your work,
under your roof.

PrivateForge is built on a single, stubborn idea: the safest place for your data is the place it already lives. We do not move it. We do not copy it. We do not ask you to trust a server we control. There is no server.

№ 01 / Architecture

Where everything actually
lives.

Most "AI app" stacks have three legs: a client, a backend, and the model. PrivateForge removes the backend. The client runs on your machine. The model runs on the provider you chose. Your work moves between those two, in your name, and no one else is in the path.

POSTURE 01◇

Local execution

The orchestrator, agents, file I/O, and audit log all run inside the app on your machine. No remote runtime.

  • macOS / Windows native
  • No container required
  • No phone-home
POSTURE 02◇

Your keys, your account

Provider keys are stored in OS keychain. Requests go directly from your machine to the provider, under your terms of service.

  • macOS Keychain
  • Windows Credential Manager
  • No proxy
POSTURE 03◇

No training on your work

We do not collect your prompts, files, or memory. Provider terms govern cloud calls; we recommend reading them.

  • Zero retention
  • No telemetry by default
  • Crash reports opt-in
POSTURE 04◇

Audit log

Every tool call, file write, and URL visit is recorded locally. Replayable. Exportable. Deletable.

  • Tool calls
  • File writes
  • Network egress
POSTURE 05◇

Connector permissions

Each connector authorizes against your account with the narrowest scope that works. You can revoke anytime.

  • Per-connector OAuth
  • Scoped tokens
  • One-click revoke
POSTURE 06◇

Optional cloud sync

Off by default. When on, end-to-end encrypted with a key only your devices hold. We cannot decrypt it.

  • E2E encrypted
  • Per-device keys
  • Off by default

№ 02 / Data flow

Where prompts go Direct to your chosen model provider. No PrivateForge server in the path.
Where outputs go To your local disk, in the project folder you specify. Nothing copied off-device.
Where memory lives Local SQLite database in your home directory. You can read it. You can edit it. You can delete it.
Where the audit log lives Local file, append-only. Rotated weekly. Configurable retention.
Where connector tokens live OS keychain (macOS Keychain / Windows Credential Manager). Never in plaintext on disk.
Crash reports Off by default. Opt-in, scoped to stack traces with PII filtered. No prompt or file content.

№ 03 / Compliance

SOC 2 Type I in progress for cloud sync. Local-only usage requires no certification — the data never leaves you.
GDPR By design. No data transfer occurs because no data is collected. DPA available on request.
HIPAA Local-only mode is suitable for PHI workflows. BAA available on Enterprise.
Sub-processors None for local mode. Cloud sync uses one storage provider, listed publicly.

№ 04 / Reporting issues

Vulnerability disclosure security@privateforge.ai · PGP key on request. We respond within 48 hours.
Bug bounty Coordinated disclosure with named credit. Cash bounty program in scoping for v1.1.
Status page status.privateforge.ai for cloud-sync uptime.

The safest server is no server.

— PrivateForge security posture, v1.0
